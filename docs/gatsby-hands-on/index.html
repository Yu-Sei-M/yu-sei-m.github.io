説明：GatsbyもReactなどと同様にコマンド一発でアプリの雛形が完成しますが、Gatsby公式に共有されているテンプレートを使えばさらに効率よくアプリの雛形を作成することができます。

指示：配信を見ながら一緒にテンプレを選ぶ作業を進めていただきます。まずは「GatsbyJS テンプレート」でググりましょう！

GatsbyJS公式のテンプレサイトを見るといろんなテンプレートが公開されていました。 今回はBlogを作りたいので、SEO対策が施され、PWA対応しているテンプレートを選びましたね。

説明：ここでは、テンプレートからGatsbyアプリの雛形を作っていきます。と言っても通常のGatsbyアプリの作成と同じコマンドを使って実行するので、何か難しい作業が必要なわけではありません。進めていきましょう。

指示：以下のコマンドを順番に１つずつ実行しましょう。

npm install -g gatsby-cli

gatsby new gatsby-blog https://github.com/maxpou/gatsby-
starter-morning-dew

cd gatsby-blog

gatsby develop

※ コマンドの実行（特に１つ目）は時間がかかる場合があります。

事前準備資料に記載している部分なので、すでに１つ目は実行している方が多いと思いますが、ここで一旦フィットネス（小休憩）を挟みます。

指示１：次のビルドコマンドを実行しましょう。

gatsby build

このbuildコマンドを実行するとbuildディレクトリが作成されます。

buildディレクトリとは、本番環境で公開するために書き出されたウェブアプリ（ウェブサイト）のディレクトリのことです。

このコマンドを実行したタイミングで、講義パートで説明した 静的サイトが作成（ジェネレート） されています。この機能ゆえにGatsbyJSは「静的サイトジェネレータ」に区分されるフレームワークです。本当はもう少し奥が深いフレームワークですが、ここではSSGの観点だけ押さえておけばOKです！

説明：プログラミングを勉強すると聞くと、多くの場合「何にどう活用するのかわからない文法を学ぶ」という切り口から入ることが多いかと思います。 今回は「結果から逆算して必要な部分をオンデマンドに学ぶ」という方法でプログラミング・アプリ開発の習得の入り口に立ってみました。

  1. テンプレサイトを開く
  2. 使えそうなもの探す
  3. テンプレ個別ページを開く
  4. 指定のコマンドをコピーして実行する
  5. 自分のエディタでテンプレを開く
  6. 中身を確認しどこをいじればいいか把握する
  7. 一箇所ずつ変更して試行錯誤しつつ自由に変更する
  8. 変更が終わったらビルドコマンドを実行する
  9. Netlifyログインしビルドディレクトリをドラッグ&ドロップする

指示：上にまとめた９つのステップについて今日のハンズオンで選んだテンプレートで何度か繰り返し作業してみると、他のフレームワークを学ぶ時にも使えるはずなのでしっかり覚えておきましょう。
