発展課題：かなり早く終わった人のみウェブアセンブリについて調べて知識を深めてみましょう。

調べてからでも調べる前でも、以下に書いた説明がシンプルにまとまっており概要を理解しやすいと思いますので、時間が余った人は一読しておくと良いでしょう。

導入：WebAssemblyはWASMとも呼ばれる、JavaScriptの産みの親であるブレダン・アイクによって提起された技術です。フロント開発はやれモダンだ最先端だとJavaScriptがもてはやされていますが、それはブラウザ上で処理を走らせることができる言語がJavaScriptだからです。その前提で、AppShellやServiceWorkerなどがハイスピードな情報表示を実現してきました。一方で、その前提を覆して高速化を実現しようとしているのがWASM（ワズム）です。

参考情報

  • WebAssembly

上述の通り、SPAもPWAも情報をより高速に表示し、より上質なユーザー体験を追求する中で生まれてきた技術や仕様です。

ウェブアセンブリー（WASM）は、端的に言えばJavaScript以外の言語のコードもブラウザで動く機械語にコンパイルする仕組みです。バイナリコードでアプリをネット上に公開するのが基本となるムーブメントが（JavaScript開発者の起案で）着実に進んでいるということです。現在のモダンフロントより一層高速に機能するアプリケーションがウェブを席巻する日も近いのかもしれません。

まとめ：JavaScript以外の言語で書かれたプログラムもブラウザで動く機械語のプログラムに変換するWASMは、ウェブ開発の言語の選択におけるJavaScript制限を取り払い、OS層でしか行われていない低レイヤの処理すらブラウザ上で行われるのが当たり前の状態が近づいています。
